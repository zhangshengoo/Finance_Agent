04 核心操作

Ingest · Query · Lint · Archive

所有写入必须通过 Skills 或 scripts；禁止直接编辑 raw/ 与 ontology/graph.jsonl。

finance-ingest

SKILL · P0

raw → wiki 两步 CoT（先分析后生成），含 SHA256 增量缓存与多格式提取。

  • 触发：「摄入财报」「ingest filing」
  • 步骤：读 raw → 讨论要点 → 写 filings-summary → 更新 companies / sectors / index / log → 追加 ontology 边

company-page

SKILL · P0

模板化创建 / 更新公司画像页 → wiki/companies/<TICKER>.md。

  • 触发：「新建公司 X」「更新公司画像」
  • 消费 filings-summary 综合输出

thesis-archive

SKILL · P0

对话中形成的投资判断 → wiki/theses/<slug>.md，支持 supersede 旧论点。

  • 触发：「存论点」「supersede 旧论点」
  • 查论点时自动过滤 status: superseded / invalidated

检索策略

QUERY · 5-STEP

默认 BM25 / grep，量大时才打开 C 类向量。

  • ① grep _index.json 找候选页
  • ② ontology traverse 扩展邻居
  • ③ [C 类] 合并向量命中
  • ④ 过滤 superseded / invalidated
  • ⑤ 全文读 · 答完询问是否归档
05 与 karpathy_gist 对照

Idea ↔ Implementation

本仓库是 karpathy「LLM Wiki」模式在金融垂类的工程加固版。下表标注：CORE 直接复刻 · EXT 金融化扩展。

karpathy 概念 本仓库落点 说明 / 增强
Raw sources（不可变）CORE raw/{filings,research,transcripts,…} 按金融语料 typed 切 9 类 + SHA256 缓存防重复摄入
The Wiki（LLM 写人读）CORE wiki/{companies,theses,…} 不是自由 markdown，而是 13 类 typed entity，每类有模板与 frontmatter
Schema 配置（CLAUDE.md）CORE CLAUDE.md + _schema.md + _areas-registry.md + _capabilities.yaml 拆成 4 份：手册 / 校验 / 归属决策树 / 能力开关

06 铁律

禁止与必须

维护一个对"杜撰数据"零容忍的投资决策档案，需要明确禁止清单。

// FORBIDDEN

禁止 · NO BYPASS
  • 修改 raw/（不可变）
  • 直接编辑 ontology/graph.jsonl
  • 创建无 frontmatter 的 wiki / thoughts 页
  • 杜撰财务数据 / 持仓数字 / 产品费率
  • 修改 .ingest-cache/ · .ingest-queue/
  • thoughts/ 写 visibility: public
  • 替用户给 thesis / idea 添油加醋

// REQUIRED

必须 · ENFORCED
  • 摄入走 finance-ingest 两步 CoT
  • wiki / thoughts 页含完整 frontmatter（wiki 含 sources[]）
  • 维护 [[wikilink]] 双向引用
  • portfolio / thesis / decision 涉及时尤其谨慎
  • ontology 写入走 scripts/graph_append.py
  • finance-research 先 Phase 0：KB 去重 + 提案确认
