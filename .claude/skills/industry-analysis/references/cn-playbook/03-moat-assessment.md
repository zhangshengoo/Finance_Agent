# §3 护城河评估（4 类 moat → 投资含义）

## 为什么

Step 5「投资含义」如果只是「最佳风险回报 / 主题 / 多空 / 催化剂」四条 bullet，容易写成行业新闻摘要。真正决定龙头能不能长期拿住超额利润的，是**护城河**——它把「这个行业现在景气」升级成「这家公司凭什么持续赢」。在写多空争论之前，先对龙头做一次护城河评级，多空逻辑就有了根。

## 4 类护城河

对每个核心龙头（通常 1–3 家），按 4 类逐项评 **强 / 中 / 弱**：

| 护城河 | 评估什么（A 股语境举例） |
|---|---|
| 网络效应 | 用户/供给飞轮强度、双边 vs 单边。如平台电商、券商交易生态 |
| 切换成本 | 技术集成深度、合同锁定、使用习惯。如工业软件、银行对公、半导体设备验证周期 |
| 规模经济 | 量大带来的单位成本优势、最小有效规模。如锂电/光伏制造、面板、快递 |
| 无形资产 | 品牌、专有数据、牌照/资质、专利。如白酒品牌、创新药专利、银行牌照、稀缺矿权 |

**Example（白酒龙头护城河）：**

```
| 护城河 | 贵州茅台 | 说明 |
|---|---|---|
| 网络效应 | 弱 | 非平台生意 |
| 切换成本 | 中 | 消费习惯/送礼场景黏性 |
| 规模经济 | 中 | 产能受基酒/年份约束，规模非核心 |
| 无形资产 | 强 | 品牌+稀缺产能+提价权，核心护城河 |
```

## 综合三要素

评完护城河，综合成 3 句：

1. **难以复制的持久优势** → 映射到上面 4 类的哪一类。
2. **难以修复的结构性弱点**（如集采暴露、技术路线被颠覆风险）。
3. **现状 vs 趋势**——护城河在变宽还是变窄。

这三句直接喂给 Step 5 的**多空争论**：持久优势 → 牛方论据；结构性弱点 / 趋势变窄 → 熊方论据。

## 落到输出 + 前端字段

- 护城河评级表 + 三要素写进 sector 分析 markdown 的 Step 5「投资含义」段。
- **多空结论上浮前端**：把「持久优势」转成 `bull[]`、「结构性弱点/趋势变窄」转成 `bear[]`（intake envelope `meta.frontend.bull/bear`，契约见 [frontend-kb-binding](../../../../docs/frontend-kb-binding.md) §2.1）。这样前端「行业分析」卡片的牛熊区有据可依，而非空泛口号。
- P0 不强求做 Beneish/Altman/Piotroski（→ P2，见 SKILL 不做清单）。
