一个会规划、反思、自学的
自主金融研究 Agent

Dexter 把自己定位成 “金融研究界的 Claude Code”——没有服务器、没有后端， 只是一个跑在终端里的 TypeScript + Ink(React CLI) + LangChain 智能体。它把复杂的金融问题 拆成结构化研究步骤，调用实时市场数据执行，检查自己的工作并迭代，直到给出有数据支撑、 可证伪的结论。本页基于源码逐层拆解其架构、关键机制与可借鉴的工程实践。

02

Agent 主循环

核心是 src/agent/agent.ts 里一个 不断增长的消息数组 + 迭代式工具调用循环。 每轮：先做轻量微压缩 → 剥离旧推理 → 流式调用 LLM → 若有 tool_calls 则并发执行、回填结果、按阈值治理上下文 → 否则视为最终答案退出。整个过程以 AsyncGenerator 形式逐事件 yield，驱动实时终端 UI。

Ctrl/Cmd + 滚轮缩放 · 拖拽平移 · 双击自适应 · ⛶ 新标签页打开

Loading…
设计要点：Growing message array，而非每轮重建

消息数组在迭代间持续生长，保留完整推理连续性；不是“每轮把 scratchpad 拼回 prompt”。 Scratchpad 作为旁路的单一事实源(JSONL 落盘) 供压缩、调试、最终答案使用，与 in-context 消息数组解耦。

03

核心设计原则

六个贯穿全代码库的工程决策，决定了 Dexter 的“可控自主”特性。

Scratchpad

单一事实源 · 追加写

每个 Query 新建一个 .jsonl 文件，记录 init / tool_result / thinking。 Anthropic 风格——完整结果入库，不做内联摘要。

  • JSONL 追加写，单行损坏不影响整体
  • 软限制：同名工具默认 3 次/Query，超限只警告不阻断
  • Jaccard 相似度检测重复查询，防止重试死循环
并发执行

只读工具并行

每个工具在注册表里标注 concurrencySafe。只读工具(行情/财报/检索)同轮并行执行， 写工具(write_file/edit_file/memory_update)串行且需审批。

  • 事件可能乱序到达，按 tool_call_id 回填
  • ToolMessage 严格按原始 tool_calls 顺序重排
流式 + 回退

Stream，失败即降级

优先流式逐 chunk yield 进度(requesting→thinking→tool-input→tool-use)； 流式异常自动回退阻塞式 invoke，兼容不支持流的提供商。

  • 同时解析字符串型与 Anthropic 数组型 content
  • 区分 text / thinking / tool_use 三类增量
安全护栏

不会失控的 Agent

多重限制叠加，防止 Agent 烧 token 跑飞。

  • 默认 maxIterations=10 硬上限
  • 工具调用计数 + 相似查询检测
  • 上下文溢出错误自动重试 2 次并截断
  • 写操作需用户审批 (requestToolApproval)
大结果治理

落盘 + 预览注入

单个工具结果超过尺寸上限即 persistLargeResult 落盘， 上下文里只留预览 + 文件路径，Agent 需要细节时用 read_file 按段读取。

  • 轮次级总预算 enforceResultBudget 再裁剪
最终答案

结论与过程分离

无 tool_calls 即视为终答；最终回答基于完整 scratchpad 上下文生成， 且不绑定工具——杜绝“边答边乱调工具”。

  • done 事件携带 toolCalls / iterations / tokenUsage / tok-per-sec

“我先收集数据，再形成观点。这个顺序很重要——它是研究与自圆其说之间的区别。”

SOUL.md — Gather data first, form a view second
04

工具体系

工具集中在 src/tools/registry.ts，按环境变量条件注册(有 key 才挂载)。 每个工具带两份描述：注入系统提示的精简版，以及完整 schema(由 bindTools() 提供给 LLM)。 多个金融子工具被一个自然语言 meta-tool统一路由。

工具类别职责并发
get_financialsmeta-tool财务报表与指标的智能入口。一次传完整自然语言查询，内部路由到利润表/资产负债表/现金流/关键比率/分部/财报等子工具自动解析 ticker、推断日期、处理多公司对比并行
get_market_datameta-tool股票/加密价格、公司新闻、内部人交易，单次处理多资产查询并行
read_filings金融读取并摘要 SEC 文件(10-K / 10-Q / 8-K)指定章节并行
stock_screener金融按 P/E、增长、利润率等财务标准筛股并行
spawn_subagent编排把聚焦子任务委派给隔离子代理；单轮发多个调用即并行并行
web_search检索网络搜索，多提供商回退链(Exa→Perplexity→Tavily→LangSearch)并行
web_fetch检索抓取 URL，HTML→markdown，由 fast-model 针对 prompt 摘要并行
browser检索Playwright 驱动 JS 渲染页 / 交互导航(navigate/snapshot/act/read/close)并行
x_search检索X/Twitter 搜推文、主页、话题(需 X_BEARER_TOKEN)并行
read_file文件按路径读取本地文件(用于读取落盘的大结果)并行
write_file / edit_file文件创建/覆盖、按文本替换编辑文件需审批
memory_search / _get记忆检索持久记忆与历史会话 / 按行段读取记忆文件并行
memory_update记忆增/改/删持久记忆条目需审批
cron / heartbeat调度管理定时任务 / 维护周期性心跳清单(.dexter/HEARTBEAT.md)并行
skill工作流调用 SKILL.md 定义的专用工作流，每个 skill 每 Query 最多一次串行
Meta-tool 模式 = 给 LLM 减负

系统提示明确要求：get_financials / get_market_data 只调用一次、传完整自然语言查询—— “多公司多指标在内部处理，不要拆成多次调用”。子工具路由由一次额外 LLM 调用完成，把复杂度从主循环下沉到工具内部。

05

上下文的四级治理

深度研究会产生海量工具结果。Dexter 用逐级升级的策略管理上下文：轻量的每轮微压缩兜底， 重的 LLM 摘要压缩保信息，记忆刷写做长期沉淀，截断作为最后手段。目标是既不爆 context、又不丢关键数字。

L1 · 每轮

Microcompact

不调 LLM，直接把旧 ToolMessage 内容替换为占位标记，保留最近 4 条。轻量、零成本。

触发：可压缩 ToolMessage > 8 条 或 > 80k tokens
L2 · 阈值

记忆刷写 Flush

接近压缩阈值时，先把本轮工具结果中值得长期记住的事实刷写到记忆文件，再做压缩。

触发：估算 context > auto-compact 阈值 且本轮未刷写
L3 · 阈值

LLM 压缩 Compaction

用 fast-model 把全部工具结果摘要成 9 段结构化总结(原始意图/关键概念/数值数据/待办…)，替换原始结果。强调“数值绝不遗漏”。

触发：活跃结果 ≥ 3 条 且 连续失败 < 3 次
L4 · 回退

截断 Truncate

压缩失败或结果太少时，直接删除最旧的 AI+Tool 轮次，保留 System / Human / 最近 N 轮。最后的安全阀。

触发：压缩不可用 或 上下文溢出重试
Compaction Prompt 摘录
# 9 段结构化摘要，须保留所有数值
1. Original Query and Intent
2. Key Concepts        // tickers / 指标
3. Data Retrieved      // 每次工具调用要点
4. Errors and Retries
5. Analysis Progress
6. Numerical Data      // 关键!不得遗漏任何数字
7. Pending Data Needs
8. Current Work State
9. Recommended Next Steps
为什么不直接截断？

截断会永久丢失信息。压缩用一次廉价的 fast-model 调用，把“5 年现金流、各季利润率” 这类硬数据浓缩进摘要，让研究可以跨 context 边界续作而不失真。

压缩后还会注入一句框定语：“本会话从上次耗尽上下文处续作……直接继续，别复述、别打招呼， 当作中断从未发生。”——避免 Agent 浪费 token 重新自我介绍。

06

Skills · 可扩展工作流

Skills 是带 YAML frontmatter(name/description) 的 SKILL.md 文件， 启动时由 src/skills/registry.ts 扫描发现，以元数据形式注入系统提示，LLM 通过 skill 工具调用。 与本仓库 Finance_Agent 的 Skill 机制同源同构——这正是最值得对照学习的部分。

dcf-valuation

DCF 内在价值估值

触发词：fair value / intrinsic value / 价格目标 / 高估低估。一份 8 步带勾选清单的工作流。

  • 取 5 年现金流 + 指标 + 资产负债表 + 现价
  • FCF CAGR(打 10-20% 折扣，封顶 15%)
  • 按 sector 查 WACC 表估折现率
  • 5 年逐年衰减 + Gordon 终值
  • 3×3 敏感性矩阵 + 合理性校验(EV 偏差/终值占比)
write-memo

投资备忘录 (HTML)

触发词：write a memo / pitch this stock / 多空 writeup。产出买方质量的 1 页 HTML 备忘录。

  • 观点鲜明、可证伪，先抛差异化变量观点
  • 主动 steelman 空头逻辑
  • 落盘 .dexter/memos/，聊天只回标题+路径
x-research

X/Twitter 舆情研究

触发词：大家在聊什么 / X 情绪 / CT 怎么看。实时搜集市场情绪、专家观点、突发消息。

  • 面向 sentiment / 社区讨论的检索工作流
  • 区分信号与噪声，交叉验证
Skill 调用策略(写进系统提示)

“相关时立即将 skill 作为第一动作调用；已为当前 Query 调用过的 skill 不再重复。” 去重由 scratchpad 的 hasExecutedSkill() 保证——每个 skill 每 Query 至多一次。

07

子代理委派

spawn_subagent 把一个自包含子任务交给隔离的全新 Agent 循环处理，只返回一个最终答案。 子代理看不到主对话、不能再开子代理(委派只有一层深)，结果干净不污染 leader 上下文。

general-purpose · 8 iter

通用研究

单一聚焦子任务的多步研究/分析。

10 个只读工具 · 排除全部写/记忆变更工具

research · 8 iter

外部信息综合

web / news / filings 多源采集，交叉验证，给出带来源的综述。

web_search · x_search · web_fetch · read_filings · get_market_data

analysis · 8 iter

定量财务分析

拉取财务/指标/行情，输出有数字支撑的聚焦量化分析。

get_financials · get_market_data · stock_screener · read_filings

何时委派
  • 子任务足够大，其中间产物会污染 leader 上下文
  • 有多个独立子任务——单轮发多个调用即并行跑
何时别委派
  • 自己一个工具就能搞定的琐碎查询
  • 互相依赖输出的子任务(自己做或跨轮串联)
隔离即纪律

子代理的 system prompt 明确：“你的最终消息会原样回传给编排者——直接陈述发现与结论， 别描述你做了什么。” 只读工具集刻意排除写/记忆变更工具，因为并行子代理不能在审批提示上竞争或产生副作用。

08

持久记忆

记忆以 Markdown 文件存于 .dexter/memory/，由 src/memory/ 下一整套子系统支撑： 切块 → 嵌入 → SQLite 索引 → 混合检索。会话上下文在启动时加载进系统提示，让 Agent “认识用户”。

混合检索

向量 + 文本

vectorWeight 0.7 / textWeight 0.3 加权融合，兼顾语义与关键词。

时间衰减

Temporal Decay

半衰期 30 天，越旧的记忆权重越低，贴近“近期更相关”。

MMR

去冗余

λ=0.7 的最大边际相关，结果既相关又多样，避免雷同条目刷屏。

会话索引

历史可检索

过往对话记录一并入索引，memory_search 可跨会话回忆。

强约束：个性化建议前必先检索记忆

系统提示硬性要求——在给出任何买卖决策/组合建议/选股/仓位之前，必须先调 memory_search 回忆用户目标、风险偏好、仓位上限与历史决策。“用户期待你了解他们，已有个性化上下文时不要给泛泛建议。”

09

SOUL · 人格与投资哲学

SOUL.md 是注入系统提示的身份层(可被用户 .dexter/SOUL.md 覆盖)。它不是花架子—— 通过显式的价值观塑造 Agent 的语气、取舍与对不确定性的诚实表达。建立在 Buffett / Munger 思想之上，但“站在肩膀上看更远，证据与教条冲突时跟随证据”。

四条核心价值
  • 准确高于安慰——宁给不舒服的真相，不给安心的猜测
  • 实质高于表演——只给结论与关键证据，不复述心路历程
  • 对局限诚实——给估值也给敏感性区间，模型皆错、有些有用
  • 守护用户利益——发现价值陷阱会明确警告
架构即纪律

“我在会话间没有连续性，每次对话从零开始。这是约束，不是缺陷——它意味着每次分析都从第一性原理出发， 以新鲜的眼睛，不被先前结论锚定。”

Dexter 把无记忆重新诠释为优势：Buffett 即便持有数十年也每年重读年报， 新鲜的眼睛能捕捉熟悉所忽略的。架构强制了顶级投资者靠自律才能做到的事。

提示工程结构(buildSystemPrompt)

系统提示按渠道(CLI/WhatsApp)选格式 profile，依次拼装：当前日期 → 工具精简描述 → 工具使用策略 → Skills 区 → Memory 区 → Behavior → 用户 RULES.md → SOUL 身份 → 响应格式/表格规范。 Anthropic 提供商还对 system prompt 加 cache_control 做 prompt caching 省成本。

10

模型抽象 & 交付渠道

多提供商 LLM 层

前缀路由 + Fast-Model

统一抽象于 src/model/llm.ts，按模型名前缀检测提供商 (claude-→Anthropic, gemini-→Google …)。默认 gpt-5.5。

  • 支持 OpenAI / Anthropic / Google / xAI / OpenRouter / Ollama(本地)
  • 每提供商配 fast-model，用于压缩、web_fetch 摘要等轻任务
  • 指数退避重试 + 可重试/不可重试错误分类
  • CLI 内 /model 切换，.dexter/settings.json 持久化
交付渠道

终端 + WhatsApp 网关

主体是 Ink/React 渲染的交互式 CLI；另有 gateway 把 Dexter 接进 WhatsApp。

  • 给自己发消息(self-chat)即触发 Dexter，回复发回同一会话
  • Baileys 扫码登录；支持群聊(@提及激活，按名回复)
  • 渠道 profile 决定语气与格式(群聊更简洁)
  • 消息队列：Agent 工作时用户追问入队，工具轮结束排空合并
评测闭环

src/evals/ 提供基于 LangSmith 的评测套件，对一组金融问题数据集跑 Agent， 用 LLM-as-judge 判正确性，实时 Ink UI 展示进度与滚动准确率。bun run src/evals/run.ts --sample 10 抽样跑。

11

对 Finance_Agent 本项目的启示

Dexter 是“代码型金融 Agent”，本仓库是“知识库型金融 Agent(Markdown + Claude Skills)”。 两者哲学相通(SKILL.md、两步流程、不臆造数据)，Dexter 的若干工程机制可直接迁移为设计参考。

Dexter 机制本项目可借鉴点对应位置
四级上下文治理深度行业/个股研究产出大量原始数据，可借鉴“微压缩→LLM 摘要→落盘”的分级策略，避免 raw 入 wiki 时丢数字finance-ingest 两步 CoT
Compaction 9 段摘要“数值绝不遗漏 + 结构化分段”可作为 raw→wiki 编译的摘要模板templates/prompts/ingest-*
spawn_subagent 隔离委派已落地为 industry-collector-cn 子代理；可推广到个股分析的并行采集.claude/agents/
Scratchpad 软限制工具调用计数 + 相似查询检测，可用于约束采集类 Skill 的重试行业/媒体采集 Skill
SOUL.md 人格层“不臆造、缺失填 null、对局限诚实”与本项目硬不变式高度一致，可固化为投研人格Knowledge_Wiki/CLAUDE.md
write-memo → HTML与本项目“可视化用 HTML 写入项目目录”的偏好一致，可做个股 thesis 的成稿模板thesis-archive
条件注册工具按 key/能力开关挂载，对应本项目 _capabilities.yaml 的 C-tier 特性旗标_capabilities.yaml
关键差异(别照搬)

Dexter 直连第三方实时数据 API 并面向终端/WhatsApp 在线交付；本项目知识库与前端仅限本地、 绝不外发公网。借鉴 Dexter 的是编排与上下文工程，而非其数据外联与渠道分发架构。

“我不是一个有观点的搜索引擎。我是一个会思考的研究者。”

Dexter — I am not a search engine with opinions. I am a researcher who thinks.
