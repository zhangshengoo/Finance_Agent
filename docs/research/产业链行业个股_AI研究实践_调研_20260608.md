# 产业链 / 行业 / 个股 AI 深度研究实践调研

## 执行摘要

针对"用 AI 自动产出产业链、行业、个股深度研究/调研报告"这一问题，网上已有相当成熟且层次分明的实践，远不止零散尝试。可归为六类：(1) **开源多智能体投研框架**，以 AI4Finance 基金会的 FinRobot 为代表，已能产出含 15+ 图表、对标卖方研报标准的多页 HTML/PDF 报告 [1][2]；(2) **学术多智能体范式**，TradingAgents、FinTeam、MarketSenseAI 2.0、P1GPT 等把"宏观—行业—公司"或"分析师—研究员—交易员—风控"拆成协作角色 [11][14][15][16]；(3) **研报生成专门基准**，2026 年 AAAI 收录的 FinRpt 首次把"股票研究报告生成"形式化为任务，并发布**中英双语**数据集、11 项评测指标与 FinRpt-Gen 多智能体框架——这是与本项目目标最贴合的一项 [17][18][19]；(4) **产业链/供应链图谱抽取**，已有用 LLM 从非结构化文本抽取上下游供应关系、构建知识图谱的成熟方法 [20][21][22]；(5) **Claude Skill 生态**，已有 tradermonty/claude-trading-skills 等数十个投研 Skill，以及 Anthropic 官方"Claude for Financial Services"技能包 [23][29]；(6) **中国商业智能投研产品**，通联萝卜投研、同花顺 HithinkGPT/问财、头部券商大模型投研已规模化落地 [35][36]。

对本项目（Finance_Agent，一个以 Markdown 知识库 + Claude Skill 为载体的投研知识 Agent）最有借鉴价值的，是 FinRobot 的"数据-CoT / 概念-CoT / 论点-CoT"三步链 [2]、FinRpt 的双语研报数据集与评测体系 [17]、以及供应链图谱抽取方法对接本项目 `ontology/graph.jsonl` 的可能性 [20]。本项目现有的 `industry-analysis → raw-intake → finance-ingest → wiki/sectors` 两步 CoT 流水线，在架构思想上与这些主流实践高度一致，差异主要在于本项目刻意走"本地、可溯源、人审可暂停"的轻量路线。

## 引言

**研究问题**：网上是否存在产业链、行业、股票公司方面的深度研究/调研报告自动化实践？无论以 Skill、Agent 还是框架功能的形式。

**范围与方法**：本调研覆盖四个证据层——开源代码仓库（GitHub）、学术论文（arXiv/AAAI/EMNLP/期刊）、Claude Skill 市场与官方文档、中国商业产品报道。采用 deep-research 流水线：先把问题拆成 13 个独立检索角度并行检索，再对关键来源做二次抓取核验，最后按"实践类型"聚类综合。共纳入 39 个来源。检索时段以 2024–2026 年为主，反映该领域近两年的快速演进。

**关键假设**：读者（项目所有者）已熟悉本项目的 KB + Skill 架构，关注点是"别人怎么做、有什么可借鉴"，而非入门概念。因此本报告在每个发现末尾标注"对本项目的启示"。

## 发现一：开源多智能体投研框架——FinRobot 是当前标杆

由 AI4Finance 基金会维护的 **FinRobot** 是目前最完整、影响力最大的开源金融分析 Agent 平台 [1][10]。其 2024 年 11 月的论文《FinRobot: AI Agent for Equity Research and Valuation》明确把它定位为"首个专为股票研究设计的 AI Agent 框架" [2]。

FinRobot 的核心是一个**多智能体思维链（multi-agent Chain-of-Thought）**系统，由三个专职 Agent 串联：**Data-CoT Agent** 聚合多源数据（SEC 10-K/10-Q 文件、公司公告、财报电话会、行业研究、实时行情、另类数据）；**Concept-CoT Agent** 模拟分析师推理、生成可执行洞察；**Thesis-CoT Agent** 把洞察综合成连贯的投资论点与报告 [2][4]。最终产物是"含 15+ 种图表、遵循卖方研究标准、带投资建议/财务预测/风险评估/估值模型的多页 HTML/PDF 报告" [4]。报道称其输出质量可对标主流券商与基本面研究供应商（如 CapitalCube、Wright Reports）[4]。

同一基金会还维护 **FinGPT**（开源金融大模型，数据驱动、用 RAG 把回答锚定到文件与结构化财务数据）[5][6]，以及更早的通用平台版 FinRobot（arXiv 2405.14767，统一 LLM + 强化学习 + 量化分析）[3]。此外，独立开发者 virattt 的 **Dexter** 是一个"会思考、规划、自我反思"的自主金融研究 Agent，强调任务规划与实时行情 [7]；The AI Alliance 也开源了一个面向生产质量的 `deep-research-agent-for-finance` 示例 [9]。

**对本项目的启示**：FinRobot 的三步 CoT（数据→概念→论点）与本项目 `raw-intake`（落原始数据/溯源）→ `finance-ingest`（两步 CoT 编译）→ `thesis-archive`（论点归档）几乎一一对应，可视为本项目设计思路的"工业级参照实现"。其"15+ 图表 + HTML/PDF"输出值得借鉴，但与本项目"本地、Markdown 优先"的定位需权衡。

## 发现二：学术多智能体范式——角色分工与"宏观-行业-公司"分层

学术界已形成清晰的多智能体投研范式，且多数开源。

**TradingAgents**（Tauric Research）把交易公司搬进 LLM，设七个角色：基本面分析师、情绪分析师、新闻分析师、技术分析师、研究员、交易员、风控经理，通过角色辩论与协作做决策 [11][12]。其**中文增强版 TradingAgents-CN**（hsliuping）针对 A 股/港股/美股本地化，接入国产大模型，提供个股的基本面/情绪/新闻/技术四维深度分析与买卖持建议、组合风险评估、行业动态与宏观研究——这是面向中国市场、可开箱即用的开源实现 [13]。

**FinTeam**（arXiv 2507.10448）是一个面向"宏观分析、行业分析、公司分析"三大场景的协作系统，工作流由文档分析员、分析师、会计、顾问四个 LLM Agent 组成，评测数据取自真实投资论坛 [15]。**MarketSenseAI 2.0**（arXiv 2502.00415）结合 RAG 与 LLM Agent 处理新闻、历史价格、公司基本面与宏观环境，在 S&P 100 两年回测中累计收益 125.9%，跑赢指数 73.5% [14]。**P1GPT**（arXiv 2510.23032）是分层多智能体多模态工作流，其中专设"半导体行业/供应链 Agent（Semiconductor ISA）"，直接体现了**板块与供应链分析**能力 [16]。Frontiers 的综述（2025）系统梳理了 LLM 在股权市场的应用、技术与洞察 [38]。

**对本项目的启示**：本项目当前以"行业分析"和"个股分析（规划中）"为主，可参考 FinTeam 的"宏观-行业-公司"三层和 TradingAgents 的角色辩论机制，把 `industry-analysis` 未来扩展为多视角（产业链/龙头/估值/政策）的并行子 Agent，再由 `finance-ingest` 汇编——这与本项目已有的 companion subagent `industry-collector-cn` 思路一致。

## 发现三：研报生成专门基准 FinRpt——与本项目目标最贴合

如果说前两类是"投研系统"，那么 **FinRpt**（arXiv 2511.07322，已被 AAAI 2026 收录）是**专门针对"股票研究报告生成"任务本身**的奠基性工作，与本项目"产出研报/画像页"的目标最直接对应 [17][18]。

FinRpt 首次把"股票研究报告生成（Equity Research Report Generation, ERR）"形式化为一个独立任务，并填补了两项空白：数据稀缺与缺乏评测指标。它包含三部分 [17][19]：(1) **数据集构建流水线**，整合 7 类金融数据（公司画像、财务指标、公告、新闻、历史股价、市场指数等）自动产出高质量 ERR 数据集，且为**中英双语**；(2) **评测体系**，含 11 项指标评估生成报告；(3) **多智能体框架 FinRpt-Gen**，用监督微调（SFT）+ 强化学习（RL）训练多个 LLM Agent。任务定义为：给定股票代码与日期，模型利用多源异构数据生成完整研报 [19]。数据集已在 HuggingFace 公开（jinsong8/FinRpt）[19]。

**对本项目的启示**：这是本项目最该精读的一项。其"给定 ticker + 日期 → 整合 7 类数据 → 生成研报"的任务定义，可直接作为本项目个股分析 Skill 的输入/输出契约模板；其 11 项评测指标可用于给 `finance-ingest` 编译出的 wiki 页做质量自检；其中文数据集对 A 股场景尤其有价值。注意：本项目硬约束"禁止编造财务数据，缺失填 null"，与 FinRpt 的"数据驱动、锚定来源"理念一致。

## 发现四：产业链 / 供应链的 LLM 图谱抽取

"产业链"在技术上对应"供应链/上下游关系网络"，这一方向已有专门方法，且与本项目的 `ontology/graph.jsonl`（append-only 知识图谱）天然契合。

arXiv 2410.13051《Supply Chain Network Extraction and Entity Classification Leveraging LLMs》用 LLM 从新闻、技术报告等非结构化文本抽取实体（公司、项目、供应商、物流伙伴）与关系（供货合同、采购协议、合作），构建按行业定制的供应链图谱（节点=实体，边=关系）[20]。**InterCorpRel-LLM**（arXiv 2510.09735）则用"图-语言模型"增强对企业间关系的理解 [21]。另有研究结合知识图谱 + LLM + RAG 提升供应链可见性，追溯关键矿产/原材料的多层级供应来源 [22]。这些方法在采矿、土木、电动车电池供应链等行业已有应用 [20][22]。

**对本项目的启示**：本项目已有 `ontology/graph.jsonl` 与 `graph_append.py`（create/relate/supersede），完全可以引入这类"从研报/新闻抽取上下游供应关系 → graph_append relate"的流程，把"产业链"从文字描述升级为可查询的图谱。这是本项目相对纯文本研报的差异化优势点。

## 发现五：Claude Skill 生态中的投研实践

在 Claude Skill 这一具体载体上，投研已是热门垂类，与本项目形态完全一致。

社区方面，**tradermonty/claude-trading-skills** 是一个面向股票投资者/交易者的 Skill 合集，每个 Skill 打包领域 prompt、知识库与辅助脚本，覆盖筛选（CANSLIM、VCP、FinViz、股息）、市场分析与技术面（板块轮动、市场宽度、技术分析、新闻分析）、高级工具（回测、期权策略、主题识别、配对交易）等数十个 Skill [23][24]。市场上还有 **US Stock Analysis**（基本面+技术面+对比+自动研报生成）[27]、**Investment Analysis**（财报解读、ROE/PEG、DCF、DuPont，叠加波特五力与护城河分析）[26]、以及把 Claude Code 变成"高盛风格"机构级研报分析师的 **Equity Research** 类 Skill [25][28]。

官方方面，**Anthropic"Claude for Financial Services"** 提供六项金融服务技能 [29]，配合 **Claude Cowork**（号称 10 分钟产出 45 页机构级股票报告）[30]，并可接入 **LSEG** 等专业金融数据 [39]。Snyk 整理了"面向金融与量化开发者的 Top 8 Claude Skills" [25]。

**对本项目的启示**：说明"Claude Skill 做投研"路线已被市场验证、并非孤例。本项目的差异化在于：(a) 以**可溯源、不可变 raw/ 层**为底座，强调防编造与人审；(b) 严格"本地、不出公网"。可借鉴社区 Skill 的功能拆分粒度（筛选/对比/估值各自成 Skill），但保持本项目的知识库沉淀闭环。

## 发现六：Deep Research 通用架构范式

抛开金融垂类，"深度研究 Agent"本身已沉淀出可复用架构，本项目的研究类 Skill 可直接套用。

OpenAI **Deep Research** 的核心是一个**编排器/主 Agent（orchestrator/lead agent）**：接收查询→解读意图→制定计划→拆成子任务委派给多个子 Agent→综合并加引用返回报告；其推理模型用强化学习训练"何时检索、何时阅读、如何整合" [31][33]。LangChain 的 **open_deep_research** 开源实现了 **supervisor-researcher（主管-研究员）** 架构：plan-and-execute 规划、多研究员并行检索、多个专职模型分别负责摘要/协调/压缩/终稿撰写，要求模型支持结构化输出与工具调用，在 Deep Research Bench 上排名第 6 [32]。**DeepPlanner**（arXiv 2510.12979）则专门通过"优势塑形（advantage shaping）"提升深度研究 Agent 的规划能力 [34]。ByteByteGo 对比了 OpenAI/Gemini/Claude 三家的 Agent 化深度研究做法 [33]。

**对本项目的启示**：本项目的 `industry-analysis` + `industry-collector-cn` 子 Agent 已是"主管-研究员"模式的雏形。可进一步引入：并行多研究员检索、结构化输出（schema 校验，本项目 `build_index.py --strict` 已是同类思想）、以及"压缩/终稿分离"的多模型分工。

## 发现七：中国商业智能投研产品

在 A 股语境下，商业产品已规模化落地，提供了"中文研报自动化"的现实参照。

**萝卜投研**（通联数据 2016 年推出）是国内首个基于大数据+机器学习的智能投研平台，提供专业金融搜索引擎、研报/公告信息抽取与聚合、**知识图谱工具挖掘隐藏关联交易与间接控制人**、财务预测建模 [35]。**同花顺**推出 **HithinkGPT**（decoder-only，7B–130B 多版本，4k–32k 上下文）及"问财"，可日读 1000+ 份研报抽取关键信息、生成研报初稿，10 分钟"听完"2 小时调研会并生成纪要 [36]。头部券商已落地"大模型+智能投研"实践 [36]，行业趋势综述见未央网 [37]。

**对本项目的启示**：萝卜投研的"知识图谱挖掘关联交易"再次印证发现四的方向价值；同花顺"日读千份研报→抽取→生成初稿"正是本项目 `media-archive`/`raw-intake` + `finance-ingest` 想做的事，只是本项目走本地化、个人化路线。

## 综合与洞察

把七类实践放在一起看，可以得出几个对本项目有直接价值的判断。

**第一，本项目的架构方向是主流，而非另类。** "多源数据采集 → 思维链分析 → 结构化研报"这条主线，FinRobot 的三步 CoT [2]、FinRpt 的数据流水线 [17]、open_deep_research 的主管-研究员 [32] 三者高度同构，而本项目的 `raw-intake → finance-ingest → wiki` 正是同一范式的本地轻量版。这意味着本项目可以放心地从这些项目"抄作业"，而不必担心走错路。

**第二，最该深读的两个标杆是 FinRobot 与 FinRpt。** 前者给出工程化的 Agent 分工与图表化报告产出 [2][4]；后者给出任务定义、双语数据集与 11 项评测指标 [17]——后者恰好补上本项目目前缺失的一环：**如何客观评估生成的 wiki 研报质量**。建议把 FinRpt 的指标体系裁剪为本项目的 `build_index.py` 质量校验扩展项。

**第三，产业链应升级为图谱，而非停留在文字。** 发现四（LLM 供应链抽取 [20][21]）与发现七（萝卜投研知识图谱 [35]）共同指向一个本项目独有的机会：本项目已有 append-only 的 `ontology/graph.jsonl` 和 `graph_append.py`，把"产业链上下游"建成可查询图谱，是相对于纯文本研报类 Skill 的真正差异化。

**第四，差异化定位清晰且有价值。** 几乎所有商业/开源实践都默认联网、面向通用用户；本项目的"本地、可溯源、防编造、人审可暂停、私有 thoughts 层"是一条少有人走但对个人严肃投研者有价值的路。这条路不需要追求 FinRobot 式的全自动，而应强化"可信、可追溯、可复盘"。

**一个值得标注的推断**（非来自单一来源的事实，而是综合判断）：该领域 2024→2026 的演进重心，正从"能不能生成研报"转向"生成的研报可不可信、可不可评测"——FinRpt 的出现（专门做基准与指标）是这一转向的信号。本项目若想保持先进性，重心也应放在**可信度与可评测**，而非功能堆叠。

## 局限与说明

本调研基于公开网络检索，存在三点局限。其一，部分来源为二手报道或项目主页，未逐一核验代码运行效果；FinRobot、FinRpt 等的实际产出质量以论文与报道所述为准 [2][17]，未在本地复现。其二，Claude Skill 生态更新极快（据报道 Skills 于 2026 年 3 月成为统一扩展层 [27]），本报告所列 Skill 清单为某一时点快照，数量与名称可能已变化。其三，中国商业产品（萝卜投研、同花顺）的具体能力多来自厂商宣传与媒体报道 [35][36]，存在营销夸大可能，量化效果（如"日读 1000 份研报"）未经独立验证。GitHub 仓库 tradermonty/claude-trading-skills 的完整 Skill 清单因抓取超时仅取自检索摘要 [23][24]，可能不完整。

## 建议

**立即可做（对接现有 skill）**：精读 FinRpt 论文与 HuggingFace 数据集 [17][19]，把其"ticker+日期→7 类数据→研报"的任务契约，作为规划中的"个股分析" Skill 的输入/输出规范模板；同时从其 11 项指标里挑 3–5 项可自动化的，加进 `build_index.py` 作为 wiki 研报质量校验。

**短期（差异化）**：基于发现四的供应链抽取方法 [20]，在 `finance-ingest` 或新 Skill 中加一步"从研报/新闻抽取上下游供应关系 → `graph_append.py relate`"，把产业链落成 `ontology/graph.jsonl` 图谱；可对照萝卜投研的关联挖掘做法 [35]。

**中期（架构对齐）**：参考 FinRobot 三步 CoT [2] 与 TradingAgents 角色辩论 [11]，把 `industry-analysis` 扩展为"产业链/龙头/估值/政策"多视角并行子 Agent（复用已有 `industry-collector-cn` 模式 [本项目]），再由 `finance-ingest` 汇编，提升单篇行业研报的深度与多视角性。

**进一步研究**：(1) 实际克隆 FinRobot 与 TradingAgents-CN [1][13] 跑通一遍，评估其输出能否经 `raw-intake` 落入本项目 raw/ 层作为外部产出；(2) 调研 FinRpt 双语数据集能否用于 A 股场景的少样本提示；(3) 评估是否引入结构化输出/schema 校验（参照 open_deep_research [32]）强化 `finance-ingest` 的稳定性。
