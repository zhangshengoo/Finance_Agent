TradingAgents-CN
用一支 AI 投研团队 分析一只股票

基于 Tauric Research 的 TradingAgents 多智能体交易框架打造的中文增强版。它把一次完整的股票研究， 拆解成一支由 10+ 个大模型智能体组成的虚拟投研团队——分析师采集情报、多空研究员公开辩论、 交易员落地方案、风控委员会三方博弈、最后由经理裁决出买入 / 持有 / 卖出的结构化决策。 底层用 LangGraph 编排，支持 A股 / 港股 / 美股，全面接入国产大模型。定位为学习与研究，不提供实盘指令。

01它到底是什么

三句话定位：一个把「投研流程」编码成「智能体协作图」的开源框架，并为中文用户做了本地化与数据/模型的全面增强。

内核 · Core

多智能体决策图

用 LangGraph 把分析→辩论→交易→风控→裁决固化为有向状态图，节点即智能体，边即信息流与条件跳转。开源核心位于 tradingagents/。

增强 · CN

中文 & 国产化

相比原版新增：中文界面与报告、A股数据全链路、国产 LLM（DeepSeek/通义/智谱）、智能新闻过滤、成本优化与统一日志。

平台 · App

企业级 Web 平台

v1.0 起从 Streamlit 迁移到 FastAPI + Vue 3，配 MongoDB + Redis、用户权限、批量分析、模拟交易、报告导出，Docker 多架构一键部署。

⚠ 定位声明 框架仅用于研究与教育，不构成投资建议，也不下达实盘交易指令。AI 预测存在不确定性，投资有风险。
02核心设计理念

不是「一个大模型答一道题」，而是「一群有角色、有立场、会辩论、能复盘」的智能体，模拟真实投研机构的分工与制衡。

① 角色专业化

把问题拆给专家

技术、基本面、新闻、社交情绪各有专员，每个智能体只带一类工具、只看一类数据，避免单模型「样样通样样松」。

② 对抗式辩论

多空 / 风控公开博弈

多头与空头互相反驳，激进/保守/中立三方风控轮流发言。对立观点强制暴露风险，再由经理裁决，抑制单边乐观。

③ 分层快慢思考

quick vs deep LLM

例行任务走 quick_think_llm（便宜快），裁决/推理走 deep_think_llm（强但贵），按节点分配算力、控制成本。

④ 记忆与反思

从历史错误中学习

每类智能体配独立向量记忆库，reflect_and_remember() 用真实收益复盘决策、沉淀经验，供后续相似情形检索复用。

03多智能体决策流水线

一次分析 = 一次图的前向传播 ta.propagate(ticker, date)。情报自上而下流经五个阶段，每个阶段把结论写入共享状态，最终收敛为一个可执行信号。

05辩论与裁决机制

两处「对抗」是框架质量的核心。轮数由配置控制，越多越充分但越贵——默认各 1 轮，可按需调高。

第一处辩论

多空投资辩论

多头 ⇄ 空头交替发言，每轮针对对方论点反驳，把利好与利空都摊到桌面。结束后由研究经理权衡输出投资计划。

  • 轮数旋钮：max_debate_rounds（默认 1）
  • 裁决者：研究经理 research_manager
  • 产物：明确的投资计划 / 倾向
第二处辩论

风控委员会博弈

激进、保守、中立三方对交易员方案做风险审视，模拟真实风控会的不同声音。由风险经理裁决，决定是否/如何执行。

  • 轮数旋钮：max_risk_discuss_rounds（默认 1）
  • 裁决者：风险经理 risk_manager
  • 产物：最终买入/持有/卖出决策
条件逻辑 「该不该再辩一轮」「辩论是否结束」这类分支判断集中在 graph/conditional_logic.py，它是 LangGraph 条件边的大脑； 历史上的「死循环」问题（v1.0.1 修复）正出在这套循环控制上。
06记忆与反思闭环

这是框架从「一次性问答」走向「能积累经验」的关键——决策完成后用真实结果复盘，把教训写进向量记忆。

存

分角色记忆库

多头、空头、交易员、风控等各持独立记忆，基于向量检索（ChromaDB）召回相似历史情形。

省

反思生成

graph/reflection.py 拿当次决策 + 真实收益，让 LLM 复盘「对在哪、错在哪」，产出经验条目。

用

回写与复用

ta.reflect_and_remember(returns) 把反思写回各记忆库；下次分析时自动检索注入，形成学习闭环。

07数据层 · 多源容错

每个市场都配「主源 → 备源 → 兜底」的降级链，任一数据商挂了自动切换；再叠加三级缓存削减重复拉取与成本。位于 tradingagents/dataflows/。

过滤 + 质量评估

  • 聚合：多源实时抓取 + 去重
  • 相关性打分：关键词规则 0–100，min_score 阈值过滤（停牌/重组/解禁加分，指数基金减分）
  • 语义增强：可选 sentence-transformers 相似度 + 中文 RoBERTa 分类
