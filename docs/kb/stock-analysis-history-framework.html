02

现状：每日独立冷跑

cold rerun model

每次运行是 (ticker, date, depth) 的纯函数——辩论 state 每次 fresh init，无隐藏跨 run 状态。连续多日分析同一只票，各日完全独立：相同的稳定数据被反复抓取、几乎相同的多空/风险辩论被从零重导，历史之间没有任何复用或联动。

DAY T-1 · 冷跑
  1. 抓行情/财报/新闻/情绪
  2. 4 分析师 LLM
  3. 多空辩论 ×N
  4. 风险辩论 ×M
  5. 裁决 → 快照 T-1
✕
DAY T · 冷跑（全重来）
  1. 又抓行情/财报/新闻/情绪
  2. 4 分析师 LLM（重跑）
  3. 多空辩论 ×N（重导）
  4. 风险辩论 ×M（重导）
  5. 裁决 → 快照 T
✕
DAY T+1 · 冷跑（再全重来）
  1. 再抓一遍稳定数据
  2. 4 分析师 LLM（再跑）
  3. 多空辩论 ×N（再导）
  4. 风险辩论 ×M（再导）
  5. 裁决 → 快照 T+1
↑ 历史报告间无 fingerprint、无 memory、无 delta —— 每个箭头都付全价 LLM，且大部分冗余

● 合理面 — 该保住的地板

correctness floor — keep
  • 可复现 / 可审计：纯函数、无隐藏跨 run 状态，历史分析永远可重导，无陈旧判断悄悄漂移。
  • 与 KB 不可变契约对齐：冷跑 ↔ raw/ 按日不可变；同日 sha256 撞 = 正确的幂等边界，不是 bug。
  • 波动层保鲜正确：情绪 (intraday)、实时行情本就该每日重算，盲缓存会出陈旧信号。
  • 数据缓存结构骨架对：行情 vs 财报已分目录 + 分 collection + 分 TTL（12h / 1h）。
  • 无错误学习：memory 在 RAM、reflect 从不调用 → 不会用未验证反思污染未来决策。"坏掉的学习环"反成安全特性

● 浪费面 — 该抬的天花板

steady-state waste — raise
  • 重复抓稳定数据：财报(季度) + >7d 历史 K 线(收盘冻结)每次重抓——缓存 KEY 嵌了滚动起止日期，换天=新 MD5=必 miss；1h/12h TTL 还过夜删昨日缓存。file_cache.py:176-185
  • 重辩几乎不变的论点：8–9 个 LLM 辩论/裁决节点（含 2 个 deep_thinking 热点 ~45-60s）每天从零重跑，即使基本面+历史新闻字节相同、只有情绪/盘中价动了。~17min 里约 10min 主成本，多为冗余
  • 丢失论点演化：每日是孤立快照，「今天 vs 昨天变没变、为什么」从不计算；day-over-day diff + 多空论点延续被丢弃，尽管 KB 已存每日报告。
  • 学习机器空转：引擎 ship 了完整 reflection→ChromaDB→get_memories 环，却半接线——读路活、写+持久从不触发。reflect_and_remember 从不调用
  • memory 检索仅语义、无 ticker/date 过滤：即便持久化，匹配的是含每日变动 market_report 的拼接 embedding，"昨天同票"不保证 top match。

04

三层历史感知框架

recommended framework

保冷跑为安全默认，把昂贵判断链门控在可验证的变化信号上，再把引擎里已 ship 却空转的 memory/reflection 环接成跨日连续层。博采三方案之长。

L1 · 事实层

每日廉价刷新 + 冻结复用

Facts — daily cheap refresh / frozen reuse
机制

行情缓存重键为原子 per-day bar(去掉滚动 start/end)、查询时范围聚合只取缺的最新日；财报按 report_period 无 TTL；情绪无条件每日重算；快照里发 fingerprints{}——对结构化 DATA payload 做 sha256（不是 LLM prose）。

复用：现有 china_stocks / china_fundamentals 目录分离 + collection 分离 + report_period 键（mongo）。
L2 · 判断层

条件重跑 · delta 模式

Judgment — fingerprint-gated, delta-anchored
机制

runner orchestrator 比对今日 vs 昨日快照的 per-layer 指纹。全稳 → 整体 skip propagate() 复用昨日 investment_plan/final_trade_decision（引擎级短路）；任一越 θ → 重跑但注昨日决策为 past_memory_str + 给 Research/Risk Manager 追加 delta 指令（"复述昨日 call、列变了什么、是否改评级"）。门控基于结构化报告指纹、非缓存辩论文本。

复用：propagation.py:48-51 空报告槽 + 已活的 get_memories 读路 + past_memory_str 槽。
L3 · 记忆与演化层

持久 + KB 演化时间线

Memory & Evolution — persist + KB timeline
机制

翻 is_persistent=True(全 6 处) + persist_directory 按 ticker；runner post-propagate 调 reflect_and_remember()；预种昨日决策；纯 Python compute_delta()(diff action/target/confidence/risk) 写进快照；KB：raw-intake 带 evolution_delta、finance-ingest 给公司页 append 「## 演化链」、kb-parse.js 在 reports[] 下拉旁渲染 day-over-day 时间线。

复用：整条 reflection.py + memory.py 逻辑不动只接线；KB 侧整链不变。
05

执行流程

8 steps · same-day + cross-day

覆盖同日重跑(引擎级短路互补 KB 的 sha256 幂等) 与跨日增量(指纹门控 + delta 模式 + 演化链)。每步标模式。

REUSEINCREMENTALFRESHCONDITIONAL
0载入昨日快照与指纹reuse
读最近一个有快照的交易日 <ticker>-<prevtradingday>.json 的 fingerprints/reports/decision。
纯读、零引擎成本；"昨天"= 上个有快照的交易日(非日历昨天，处理周末/停牌)；首跑/新票 → baseline=true 优雅降级为全 fresh。
1同日重跑守卫conditional
若 <ticker>-<today> 已有快照且输入指纹一致 → 秒级短路返回既有结果。
镜像并互补 KB 的 raw 不可变 sha256；盘中指纹变(价/情绪动)则继续，delta 锚仍取上个交易日(不基于同日半成品)。
2事实层刷新 (L1)incremental
原子缓存取数：复用冻结历史 bar + 季度财报；只抓最新 bar + 新闻增量；情绪无条件重算。算今日 per-layer DATA 指纹。
情绪子步 = fresh 永不复用；财报子步 = reuse report_period 不变时。
3指纹门控conditional
逐层比对今日 vs 昨日指纹，按阈值分类 stable/changed（θ_price=收盘>1% 或新交易日；θ_news=新文>0；情绪恒变；财报仅 report_period 进位才变）。
默认全保守(宁重算不错复用)；主开关 TA_HISTORY_AWARE 默认 OFF；指纹逻辑可对存档快照离线测试，不必重跑 17min 引擎。
4a判断复用路径（4 层全稳）reuse
整体 skip propagate()。复用昨日 investment_plan / trader_plan / final_trade_decision + 结构化决策字段(Signal Processor 不重跑)。
这里省下约 10min 辩论/裁决成本(含 2 个 deep_thinking 节点)；输出与昨日 call 一致——合理，因无实质输入变动。
4b判断 delta 重跑路径（任一变）incremental
把 L1 报告注入空 state 槽跑 propagate()；注昨日决策为 past_memory_str；给 Research/Risk Manager 追加 delta 指令；Signal Processor 仅重抽变化的决策文本。
delta 模式让 deep-LLM 做"相对昨日变了什么"而非"从零合成"，省 token/延迟；防锚定偏差：强制改评级规则(关键指标越带必 re-rate)。
5持久 memory + 反思incremental
调 reflect_and_remember()；ChromaDB 落盘持久；预种今日决策供明日 get_memories 锚。
标注限制：这是"论点连续"而非"已验证 P&L 学习"(returns_losses 当前是 placeholder)，直到接真实收益指标；需 prune/TTL 策略(引擎无 memory 维护)。
6–7确定性 delta + KB 演化链 + 前端时间线incremental
纯 Python compute_delta(今,昨) → evolution_delta 写快照；raw-intake 带过(sha256 幂等)；finance-ingest 给 wiki/companies append「## 演化链」(最新视图字段仍 merge-latest)；kb-parse.js 在 reports[] 下拉旁渲染 day-over-day 时间线。
零 LLM、全可审计；日键 = 每日一个演化节点；复用整条 raw→wiki→前端链；严格 localhost(不上公网)。

诚实清单 · 落地前必读

honest caveats — defaults conservative, validate offline
  • 默认 OFF：主开关关；指纹门控逻辑先在 temp KB 对存档快照离线验证，遵守"别为验证重跑 17min 引擎"。
  • judge-only 重入未验证：L2 最大风险；先用"仍跑廉价分析师、只省辩论链"作安全退路，runtime 验证后再上全短路。
  • 指纹键于数据非 prose：LLM 报告文本会 drift，门控必须对缓存层结构化 payload 取指纹。
  • memory 是论点连续、非 P&L 学习：returns_losses 仍是 placeholder；接真实收益前不要宣称"已验证学习"。
  • 持久化要翻 6 处 + 加 prune/TTL：引擎无 memory 维护，长期需淘汰策略防 ChromaDB 膨胀。
  • 同日 vs 跨日分治：同日靠引擎短路 + KB sha256 幂等；跨日靠指纹门控 + delta + 演化链；delta 永远锚上个交易日。
