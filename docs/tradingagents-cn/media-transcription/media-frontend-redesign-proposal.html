Finance_Agent · 媒体转录前端 v2 · 设计方案（待评审）

UP 媒体转录前端 v2 — 观点演化（stance-as-data）

把「观点 / 立场」做成带时间戳、可溯源的一等公民数据，支撑核心交付：某标的 / 方向（黄金 · AI · 美债）× 时间 → UP 主观点与仓位演化图，并扩展 UP 主维度功能。本方案经四路并行调研 + 三方案评审 + 对抗式查漏，所有承重事实已对照真实文件核对。

决策 先采集第二个 UP 决策 混合权威：有 wiki 以摘要页为准 决策 演化图默认仓位/止损轴 · 可切多空泳道 决策 文档输出 HTML

01目标与核心交付

现状：媒体视图是「左轨 raw 概要 + 右轨 wiki 深度」的双轨时间线，但观点散落在 TL;DR 文本与操作表里，无法沿时间追踪某个标的上的立场变化。v2 的目标是让观点可结构化、可聚合、可画图、可回溯。

核心交付

个股 / 方向 × 时间的观点演化图：仓位动作（建仓→加仓→减持）与目标/止损位迁移随时间展开，多空作背景带。

数据基石

预览模版加一个机读 stances[] 块，把"立场"从散文里提炼成结构化、可溯源记录。

UP 维度扩展

UP 主画像（标的云 / 立场雷达 / 持仓快照）、跨 UP 主同标的对比（采到第二个 UP 后激活）。

已核对的工程事实（决定可行性）

  • build_timeline.py 的 fm_list 只匹配单行 key:[a,b,c] → YAML 块序列对它物理不可见，加 stances: 块后旧 timeline.json 逐字节可复现（零破坏，可 diff 验证）。
  • kb-parse.js 的 frontmatter 解析无法读对象列表 → stance 必须经 timeline.json 以原生 JSON.parse 进前端。
  • _schema.md 已有 stance: long|short|watch|avoid 与 factual/observation/opinion/trade-action 四档 provenance → 直接对齐，不另造体系。
  • 前端星图 SVG 基建（圆 / 路径 / viewBox / 图例 toggle / focusNode）可整套复用来画演化图，守零编译。

02现实约束核对（对抗评审发现 · 已反映进设计）

这些不是假想风险，是逐一 grep 真实数据后确认的硬约束。每条都给出本方案如何回应。

RC-1只有 1 个 UP 主
raw 下仅 战国时代_姜汁汽水 一个目录，跨 UP 共识 / 多线对比 n=1 时退化为单线，无法验证。
→ 决策：先采集第二个 UP（M0 前置任务），让跨 UP 功能一开始就有真实第二数据源。
RC-2同一标的 8+ 种写法
黄金语义簇真实变体：金银(70×) / COMEX黄金 / Gold / 贵金属 / 纽约金期货 / 黄金（纽约期货金）…；"金银"是黄金+白银复合词，并入任一即臆造。仅去括注解决不了。
→ 三段式 canon_target：① raw-preview 产出端强制规范名；② 前端保守显式别名表（精确匹配）；③ 复合/泛指词黑名单（金银 / 贵金属 → 只进话题云，不参与单标的归一）。验收：grep timeline.json 确认黄金簇并轨成 1 个、且"金银"未被错并。
RC-3真实信息在止损迁移，不在多空翻转
demo 黄金/白银全程 long、美债全程 short，50 条里 long↔short 翻转 0 次。真正在变的是止损 4665→4600→4480→4200 与仓位 建仓→加仓→减持。多空泳道对这批数据近乎平线。
→ 决策：演化图默认"仓位/止损迁移轴"，可切多空泳道；changepoint 检测从"多空翻转"改为"仓位动作变化 / 止损方向反转"。
RC-4mentions 是词表命中（含噪）
05-14 视频 mentions 含"黄金"，正文却注明"仅作货币体系背景提及，无价位判断"；还混入恰饭软广词。用 mentions 算"关注度"会被严重高估。
→ 数据层物理区分两口径：频率 / 共识 / "被几位 UP 看"只用 stances[]（真实立场）；mentions 仅渲染为"话题云"并标注"启发式·含背景噪声"。
RC-5恰饭软广污染抽取
05-14 正文有完整"海力生鱼油"带货段；2 条动态 ad_count:1。一步 CoT 读全字幕产 stance 时，广告里的词/数字可能被误抽。
→ raw-preview v0.3 契约硬规定广告段不抽进 stances[]；ad_count>0 的条目 stance 打 from_ad_risk 标记供前端过滤。复用已有 ad_count 字段。
RC-6动态无 seg 锚 · 2 视频无 preview
42 份动态全无 ^seg-N 锚（动态无"字幕分段"段），而 demo 关键点 05-07/06-02/06-09 恰是动态；8 视频里 06-04/06-08 无 preview（raw 有 JSON），黄金序列在此出现空档。
→ seg 契约按 kind 分支：视频用 ^seg-N、动态用动态 id / #dyn-k；缺失则 seg=null 不臆造。06-04/06-08 在 UI 显式标"待 preview·无立场数据"，不留无解释空档。
RC-7stance 三源漂移风险
同一条立场可能同时存在于 preview 的 stances[]、wiki 的 ### Trade Actions、正文"关键观点 callout"——谁权威没定，长期会漂。
→ 决策：混合权威（见 §3）：有 filing-summary 的条目以 finance-ingest 产出为权威并进 ontology；无 wiki 的用 preview.stances[] 兜底；正文 callout 与 stances[] 同一步 CoT 产出、内容必须一致。

03数据模型：stance 一等公民 + 混合权威数据流

stance = 一条带时间戳、可溯源的主观立场记录。它有两个"产地"，按是否已摄入 wiki 决定权威方——既守 CLAUDE.md 的 raw→finance-ingest→wiki 数据流，又让未摄入的条目也能进演化图。

04预览模版 v2（向后兼容 · 真实示例）

只在 frontmatter 加 stances: 块，正文结构与所有锚点原样不动（不破坏 extract_tldr / extract_up_actions / extract_mentions）。下例所有值均取自 2026-05-31 BV1y4VJ6YEbr 真实正文，未臆造。

# frontmatter 末尾、mentions 之后、闭合 --- 之前新增（块序列对旧解析不可见）
generator: raw-preview v0.3
stances:
  - target: 黄金
    kind: macro
    stance: long
    conviction: high          # 正文"回到5000~5200可能性非常高"
    target_price: '5000-5200'  # 原文字符串，不解析成数字
    stop_loss: '4600'          # 操作表第4列"回踩跌破4600"
    risk: 回踩后跌破4600且无法收回则破位   # 第5列(当前被丢弃)
    horizon: long
    is_trade: true            # 命中"加仓"
    provenance: trade-action  # 强制字段：自述操作
    seg: '^seg-1'
    src_quote: 黄金盘中冲破4600 我们在此位置选择加仓
  - target: 白银
    kind: macro
    stance: long
    stop_loss: '72'
    is_trade: true
    provenance: trade-action
    seg: '^seg-1'
  - target: 美债
    kind: macro
    stance: short
    target_price: null      # 宏观方向无UP自报目标 → null，不臆造
    is_trade: false           # 纯观点
    provenance: opinion       # 主观论点
    seg: '^seg-4'

兼容 & 迁移（三层保证）

  • 旧解析零改动仍跑通；无 stances 块时旧 timeline.json 逐字节可复现（diff 验证）。
  • 无块时全链路降级安全：extract_stances 返回 []、演化图为空，不崩。
  • 增量回填：一份份补，不必一次重做；按标的提及频率排序。

build_timeline 增量（纯 stdlib）

  • extract_stances() 块解析器（~25 行），补 date/src/preview 写入 item.stances[]。
  • canon_target() 去高亮 + 别名表 + 复合词黑名单（RC-2）。
  • extract_up_actions 补抽第 2 列 position、第 5 列 risk。

08风险与缓解

双源漂移：stances[] 与正文 callout / filing-summary 同概念多处 → 同一步 CoT 产出 + 混合权威单一家 + 可选 lint 比对 target 集合一致性。
target 归一漏并：canon 启发式 → 显式别名表（精确匹配优先）+ 复合词黑名单 + 回填后 grep 验收。
AI 字幕数值误差：target_price/stop_loss 源于 ai 字幕（已见"川子/卧室"误转写）→ source_confidence 弱化显示 + hover 必带 src_quote 供人工核对。
纯手拼 SVG 坐标易错：区间带/台阶线/日期映射 off-by-one → M2 用真实黄金序列目测校验；解析失败点退化散点不连线（降级不崩）。
误读为行情图：UP 自报价≠成交价 → 默认仍含强标注 + 全局"主观观点/非投资建议"分区 + 与 wiki 深度物理隔离。
回填成本与数据稀疏：50 条多为低信息量 → 分波回填、demo 序列优先、低信息量动态留降级。
性能（live 全量重画）：回填+多 UP 后 → stanceIndex 内容 hash 缓存 + 按选中标的懒渲染（M2 即写）。

09决策记录（本轮已确认）
