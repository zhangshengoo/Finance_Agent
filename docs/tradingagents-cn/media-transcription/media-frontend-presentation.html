03知识内容来源与权威

五层来源各产出什么、谁是"立场"的权威，以及两个必须分清的统计口径。

①采集 · raw JSON

B 站字幕 / 元信息落不可变源。提供时间轴权威（published_at）、标题、字幕全文。缺字段留 null，不臆造。

②预览 · raw-preview

读字幕提炼：TL;DR、mentions（话题，启发式含噪）、UP操作表、stances[]（v2 机读立场，含 provenance/target_price/stop_loss）。同一步 CoT 同时产人读与机读，内容一致。

③聚合 · timeline.json

build_timeline.py 把 ① 权威字段 + ② 提炼结果拼成前端唯一契约，并抽 item.stances[]、算 is_trade、归一标的（canon_target）。派生镜像，可重建。

④摄入 · finance-ingest

两步 CoT 编 filing-summary + 进 ontology。立场权威方：有 wiki 的条目以此为准（可全库检索）；无 wiki 的退 ② 兜底。

混合权威：立场以谁为准

有 filing-summary 的条目 → 以 finance-ingest 产出的结构化立场为权威（### Trade Actions / ontology OPINES_ON），前端实体页深度轨优先读它。

无 wiki 的条目 → 以 preview.stances[] 兜底进演化图。

正文"关键观点 callout"与 stances[] 同一步 CoT 产出、内容必须一致 → 单一权威家，消除三源漂移。守 CLAUDE.md 的 raw → finance-ingest → wiki 数据流。

两个口径必须分清（对抗评审 RC-4）：「关注度 / 共识 / 被几位 UP 看」只能用 ② 的 stances[]（真实立场）；mentions 是词表命中、含背景提及与恰饭软广噪声，仅作话题云，绝不参与频率/关注度统计。实测：05-14 视频 mentions 含"黄金"，正文却注明"仅作货币体系背景提及，无价位判断"。
