02字幕获取策略 · 优先级链

B 站视频的字幕覆盖率参差不齐——部分 UP 主全无字幕轨（如靳卫萍老师 ~80% 视频），部分有 AI 字幕。 采集脚本按下图三层顺序尝试获取文字，越靠后越耗时也越耗费 API 额度， 前一层成功则不继续。

🏛️
Tier 1

B 站官方 API

fetch_bilibili.py · bilibili-api-python 实装

调用 /x/player/wbi/v2（WBI 签名）→ get_subtitle(cid=cid)，拉取视频字幕轨列表。优先选 cc-zh（UP 主手工上传），其次 ai-zh（B 站 AI 自动生成）。速度约 1 s，质量最佳，无需下载音频。

cc-zh 人工字幕 优先 ai-zh AI 字幕 次选 需 BILIBILI_SESSDATA ~1 s / 条
Tier 1 无字幕轨（subtitle_source: none）→ 尝试 Tier 2 ↓
☁️
Tier 2

bcut-asr · B 站必剪云端 ASR

SocialSisterYi/bcut-asr · 免费 · 无 GPU 按需

把音频 URL 或本地文件提交到 B 站自家「必剪」ASR 接口，返回带时间轴的字幕。免费、无需 GPU，中文识别质量接近官方 ai-zh。缺点：维护薄弱，偶有 412 上传报错，使用前先实测。

免费无额度限制 带时间轴 需先下载音频 偶发 412 风控

调用（示例）：

pip install git+https://github.com/SocialSisterYi/bcut-asr.git
# 提交任务
asr = BcutASR(audio_url)      # 或 BcutASR(filepath=path)
asr.upload()                   # 如报 412 换 Tier 3
result = asr.result()
print(result.get_srt())
Tier 2 失败或质量不足 → 兜底 Qwen3-ASR ↓
🧠
Tier 3

DashScope Qwen3-ASR-Flash · 兜底

transcribe_bilibili.py · Third_Party/bili2text 实装

当前两层均无法获取字幕时，付费调用 DashScope Qwen3-ASR-Flash 进行云端 ASR。 通过 bili2text（Third_Party/bili2text/，已集成 DashScopeQwenTranscriber）封装： ffmpeg 解码音频（绕开 librosa Intel Mac 兼容问题） → 超 170 s 自动切块 → 并行 API 调用 → 幻觉过滤 → 合并全文。

消耗 DASHSCOPE_API_KEY Intel Mac 可用 需先下载音频 长音频自动分块

调用（Skill 脚本）：

DASHSCOPE_API_KEY=sk-... \
uv run --project Third_Party/bili2text \
  python3 .claude/skills/media-archive/scripts/transcribe_bilibili.py \
  --audio-file /path/to/audio.m4a \
  --bvid BV1xxxxxx \
  --kb-root Knowledge_Wiki \
  --prompt "金融 A股 半导体"    # 可选：领域术语提示，提升准确率

输出：更新 raw JSON 的 subtitle / subtitle_source: asr-qwen3-asr-flash 字段， 或在 raw/transcripts/bilibili/_asr_only/ 创建 skeleton JSON。

任一层成功 → subtitle_source 写入 raw JSON ↓

三处模版（决定前端能渲染出什么）

预览模版 ② raw-preview

frontmatter：mentions: {companies / indicators / macro / sectors} · subtitle_chars
正文：> [!abstract] TL;DR 要点 callout
操作表：固定表头 标的｜当前位置｜动作｜目标/止损｜风险/失效条件
署名：generator: raw-preview

③ 聚合脚本严格按这套结构解析；没有预览的源 → 要点为空，前端显示「待预览生成」。

摘要页模版 ④ finance-ingest

frontmatter：title · summary · sources[]（指回原始文件，前端靠它对齐）· content_quality · related[]
正文：「核心结论」+ 固定段「来自媒体源的观察与观点（非事实）」（Opinions / Trade Actions）

前端取 summary 作深度摘要、sources[] 作匹配键；schema 由知识库校验脚本把关。

时间线 schema ③ build_timeline

顶层：uploader · uid · count · n_videos · n_dynamics · items[]
每条：date · kind · title · tldr · mentions · up_actions · is_trade · stats · src · preview · link

这是前端读取的单一数据契约（派生镜像，可随时重建，不写 wiki）。

前端不持有模版 ⑤ live

浏览器只做两件事：① 读 timeline.json 渲染左轨；② 读 filings-summary 按 sources[] / BV 号对齐右轨深度。
不预编译、不生成中间数据，所有内容都源自上游模版产物。
