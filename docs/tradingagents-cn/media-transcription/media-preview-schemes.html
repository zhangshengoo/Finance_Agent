优
  • 立场天然成表 = 演化图零转换、最稳
  • 跨视频结构一致、可预测
  • 固定 key 解析最稳健
  • 事实(宏观/轮动)与观点(立场)分栏清晰
劣
  • 最不灵活，杂谈视频可能有空栏目
  • "树感"弱（是栏目不是树）

推荐 C（固定栏目·卡片化）：核心交付是观点演化图，它最需要"立场是稳定、一等、跨视频一致的结构"——C 把 stances 直接做成标的立场表，演化图零转换、最不易漂；固定 key 解析也最稳。视觉上用卡片化栏目即可保持简洁，并不输 B。
其余两套的定位：A 适合信息特别密、需要深层次的视频，可作为 C 的"展开看全部要点"可选深入态；B 是 C 的轻量视觉变体（去掉固定栏目约束）。
若你更看重"杂谈/非财经视频也能套"，则选 B（灵活+简洁，牺牲跨视频一致性）。请确认选哪套，我据此重建 demo 展开区 + 定 preview v0.3 模版 + build_timeline 解析。
