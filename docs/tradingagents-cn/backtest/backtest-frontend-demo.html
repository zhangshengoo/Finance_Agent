决策 × horizon 反思表

对每个 snapshot 决策按多个 horizon（5/20 日）算固定窗口前向收益，各写一条教训。孤立单 snapshot 即可跑、不重跑引擎、无前瞻偏差——这是它能成为「今天就能验证效果」的首版核心的原因。停牌 / 评估日未来 → non_evaluable 跳过；持有默认跳过。

03实现逻辑：M1 单路记忆环（首版）

首版只有 M1 一条路径，复用 TradingAgents-CN 原生设施，不另起系统。流程一条线：snapshot 决策 → 真实收益 → 写 5 桶（A）→ 次日召回（B）→ 新决策。前端读的是 runner 导出的教训文件。

关键不变量：① A 写的桶 = B 读的桶——persist dir / namespace=ticker / text-embedding-v4 逐项一致。② DashScope 仅做 Embedding，reflect 的 quick-LLM 走 claude-max-proxy:5678。③ 写前探活 防零向量；幂等台账记 (ticker,date,horizon) 防重复写。④ 黄色虚线的教训回读导出是首版唯一需新增的一步。
