① 单股分析 前端配置 异步任务

前端 SingleAnalysis.vue 收集配置 → POST /api/analysis/single 拿 task_id → 5 秒轮询状态/进度 → 完成后取完整报告与决策。

前端可配置项

  • 股票代码 + 市场：A股 6 位 / 美股字母 / 港股 4-5位或 .HK，自动识别校验
  • 分析日期：禁未来日期
  • 分析师团队（多选）：市场 / 技术 / 基本面 / 风险 / 社媒(A股禁) / 研究管理者
  • AI 模型：快速模型（市场/新闻/基本面）+ 深度决策模型（综合决策/风控），按深度推荐
  • 分析选项：情绪分析开关 · 风险评估开关 · 语言（中/英）

任务生命周期（异步）

1 · 提交POST /api/analysis/single → 生成 task_id（UUID），写 analysis_tasks，立即返回 pending
▼
2 · 后台执行FastAPI BackgroundTasks → SimpleAnalysisService.execute_analysis_background() → 组装配置 → 调引擎 TradingAgentsGraph.propagate(code,date,cb)
▼
3 · 进度回调引擎 progress_callback → Redis(RedisProgressTracker) + Mongo(progress/current_step) + WebSocket 推前端；内存 MemoryStateManager 快查
▼
4 · 落库报告写 analysis_reports，任务置 completed/100% 并存 result 到 analysis_tasks
▼
5 · 取结果GET /tasks/{id}/result → 内存优先，级联 reports→tasks.result，回传决策+多 tab 报告
前端：idle → running → completed/failed，进度条 + 当前步骤 + 已用/剩余时间；完成后展示决策卡（买/持/卖·目标价·置信度·风险分）+ 多标签报告（技术/基本面/市场/情绪/新闻/风险）。

② 模拟交易（Paper Trading）5 接口 · 6 集合

虚拟资金的完整交易闭环，支持 A股/港股/美股三套独立币种账户。撮合直接用最新行情即时成交（仅市价单），盈亏/手续费/T+1 全部在 app/routers/paper.py 内计算后落 MongoDB。

子功能矩阵（已实现）

子功能说明
多币种虚拟账户CNY/HKD 各 100万、USD 10万；现金 + 已实现盈亏分币种
下单建仓市价买入，校验现金，扣款，加权平均成本更新持仓
平仓/卖出校验可用数量(T+1)，计算已实现盈亏，回补现金
持仓管理数量/可用/冻结、均价、最新价、市值、浮动盈亏
浮动盈亏(最新价−均价)×数量，账户查询时实时算
已实现盈亏卖出时 (卖价−均价)×量，累加到账户
手续费佣金+印花税(A股卖)/交易费(港)/SEC费(美卖)，最低 5 元
T+1 规则A股当日买入不可卖（聚合当日 buy 量算可用）；港/美 T+0
订单 / 成交流水paper_orders + paper_trades，永久留痕
账户重置清空持仓/订单/现金，二次确认不可恢复
分析联动下单可带 analysis_id，关联分析报告可追溯
暂未实现（可扩展）：收益曲线、排行榜、自动/半自动下单、风险度量(VaR/Sharpe/回撤)、限价单/止损单、跨币种汇率换算。

支撑功能 队列 / 同步 / 系统

批量队列系统

即 API 描述里的"批量队列"。自研 Redis FIFO 队列（非 Celery）。

  • 结构：qa:task:{id}(HASH) + qa:ready(LIST)
  • 消费：app/worker.py 异步拉取
  • 并发：用户级 3 / 全局 50
  • 可见性超时 5min 防僵尸；重试 3 次
  • 进度经 Redis PubSub → SSE 推前端

数据同步

APScheduler 调度，多源故障转移 Tushare→AKShare→BaoStock。

  • 基础/行情/日K/财务/新闻分别定时
  • 行情盘中 */5~*/30 分钟入库
  • 新闻每 2h（仅自选股）
  • worker/*_sync_service 各源适配
  • 速率限制 rate_limiter（Tushare 分级）

选股筛选

  • 维度：行业/市值/PE/PB/ROE/换手/技术指标(MA/RSI/KDJ/MACD)
  • Mongo 视图 stock_screening_view
  • Redis 缓存 30min
  • 结果可转批量分析

鉴权 & 配置

  • JWT(HS256)+bcrypt(12轮)
  • access 60min / refresh 30d，前端自动刷新
  • 配置 SOT：文件/env > Mongo，可热重载
  • LLM 厂家/模型/Token、数据源 Token 在线配

数据接口

  • stock-data / stocks：行情/基本面/列表
  • financial / news / historical / social-media
  • multi-market：跨市场搜索
  • 港/美股按需取+缓存（非定时）
  • model-capabilities：模型等级/推荐

系统/运维

  • scheduler(32)：定时任务增删暂停触发
  • operation-logs：中间件全量埋点审计
  • usage-statistics：Token/成本统计
  • database：状态/统计/备份/导入导出
  • cache / logs / notifications

实时推送机制 轮询 + WS + SSE

5s 轮询（主）

任务中心/单股分析对进行中任务每 5 秒 GET .../status 拉进度，最稳。

WebSocket

/ws/task/{id} 推分析进度；/ws/notifications 推通知/预警，ConnectionManager 按 user_id 管理。

SSE + Redis PubSub

/api/stream/task/{id}/progress 事件流（connected/progress/heartbeat），队列任务进度经 PubSub 频道转发。
